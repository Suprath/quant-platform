# KIRA: High-Frequency Quantitative Trading Pipeline
## Complete Implementation Blueprint & Documentation

---

## Part 1: The Context — What Are We Building?

The Indian retail trading landscape is dominated by 1970s technical analysis (Moving Averages, RSI, MACD). These indicators lag the market and fail to capture the actual mechanics of price discovery: **Order Flow** and **Institutional Footprints**. 

**KIRA** is an institutional-grade, zero-latency quantitative trading engine designed explicitly for the Indian equities and derivatives market (NSE). 

Instead of predicting prices, KIRA **detects hidden alpha** using stochastic calculus. It is built to exploit the structural edge of the Indian market: retail traders irrationally chasing momentum in F&O, while institutional "smart money" leaves massive, detectable footprints in Level 2 order books and End-of-Day (EOD) delivery volumes.

**The Objective:**
Build a system capable of tracking 1,500 symbols live via the Upstox API, processing up to 300 million ticks per second, and executing dynamic mathematical models (Kalman Filters, Hawkes Processes) without succumbing to Python's inherent latency bottlenecks.

---

## Part 2: The Python + C++ Hybrid Architecture

To achieve microsecond latency while maintaining rapid strategy development, KIRA uses a **Bifurcated Hybrid Architecture**. 

Python is too slow for tick-by-tick microstructure math due to the Global Interpreter Lock (GIL) and memory allocation overhead. C++ is too rigid for rapid portfolio iteration and API management. We combine the best of both worlds.

### How the Hybrid Approach Works

1. **The C++ Core (The Hot Loop):** 
   Written in C++, this engine pre-allocates a contiguous block of memory (a Flat Buffer or `std::vector` of Plain Old Data structs) for all 1,500 symbols. When a tick arrives from the broker, C++ processes it in $O(1)$ time, updating the mathematical state entirely inside the CPU's L1 Cache. 
2. **PyBind11 & GIL Management:** 
   The C++ engine is wrapped into a Python module using `pybind11`. Before the tick loop starts, the C++ engine *explicitly releases the Python GIL*. The Python interpreter goes to sleep, allocating 100% of CPU resources to the C++ math.
3. **Zero-Copy Memory Handoff:** 
   Once the C++ engine detects a breakout or finishes processing a "time window," it re-acquires the GIL and wakes Python up. It passes the memory array of 1,500 symbols directly to Python as a **MemoryView**. Python (via NumPy) reads this data instantly with *zero copying or serialization*.
4. **The Python Orchestrator:** 
   Python uses vectorized NumPy operations to calculate portfolio sizing (Kelly Criterion), check risk parameters, and send the REST API execution requests to Upstox.

---

## Part 3: The Mathematical Engine & Implementation

The core of KIRA replaces traditional indicators with **Recursive State Estimators**. These equations are implemented as $O(1)$ structs in C++, meaning they only need the *current tick* and the *previous state* to calculate, requiring no historical lookback loops.

### 1. The Kalman Filter (Extracting True Alpha)
**The Concept:** Stock returns are 99% random walk noise and 1% true edge (Alpha). The Kalman filter dynamically adjusts to market volatility, ignoring noisy price spikes and extracting the true, hidden trend.

**The Formula:**
*   **Predict:** $P_t = P_{t-1} + Q$  *(Update Uncertainty)*
*   **Kalman Gain:** $K_t = \frac{P_t}{P_t + R_t}$ *(How much to trust the new tick vs our prediction)*
*   **Update:** $\hat{\alpha}_t = \hat{\alpha}_{t-1} + K_t \cdot (\text{Actual Return} - \text{Expected Drift} - \hat{\alpha}_{t-1})$
*   **Uncertainty Update:** $P_t = (1 - K_t) \cdot P_t$

**Implementation in C++ (`kira_engine.h`):**
```cpp
struct KalmanState {
    double alpha_hat = 0.0;
    double P = 1.0;     // Internal Uncertainty
    double Q = 1e-5;    // Process Noise
    double R = 1e-2;    // Measurement Noise (Provided by GARCH)

    inline void update(double actual_return, double expected_drift) {
        P += Q;
        double K = P / (P + R);
        double surprise = actual_return - expected_drift - alpha_hat;
        alpha_hat += K * surprise;
        P *= (1.0 - K);
    }
};

struct CusumState {
    double C = 0.0;
    
    inline bool update(double signal, double variance, double threshold) {
        double log_likelihood = (signal / variance) - (signal * signal / (2 * variance));
        C = std::max(0.0, C + log_likelihood);
        if (C >= threshold) {
            C = 0.0; // Reset after firing
            return true;
        }
        return false;
    }
};

def download_nse_bhavcopy():
    # Format date (e.g., 24032026 for DDMMYYYY)

    }
    
    print(f"Downloading Bhavcopy for {date_str}...")
    response = requests.get(url, headers=headers)
    
    if response.status_code != 200:
        print("Bhavcopy not published yet or download failed.")
        return
        
    with open("bhavcopy.csv", "wb") as f:
        f.write(response.content)
        
    df = pd.read_csv("bhavcopy.csv")
    df.columns = df.columns.str.strip() # Clean headers
    
    # Filter for Equities (EQ)
    df_eq = df[df['SERIES'] == 'EQ'].copy()
    
    # Calculate Institutional Conviction Score (Delivery %)
    # Prevent division by zero
    df_eq['TOT_TRD_QTY'] = pd.to_numeric(df_eq['TOT_TRD_QTY'], errors='coerce')

import asyncio
from questdb.ingress import Sender, IngressError

async def questdb_persistor_worker(tick_queue: asyncio.Queue):
    """Consumes parsed ticks and writes to QuestDB via ILP port 9009."""
    conf = 'tcp::localhost:9009;'
    
    with Sender.from_conf(conf) as sender:
        while True:
            # Wait for a batch of ticks
            ticks = await tick_queue.get()
            try:
                for tick in ticks:
                    sender.row(
                        'ticks',
                        symbols={'instrument_token': tick['instrument_token']},
                        columns={
                            'ltp': float(tick['last_price']),
                            'volume': int(tick['volume']),
                            'oi': int(tick.get('oi', 0))
                        },
                        # QuestDB expects time in nanoseconds
                        at=int(tick['exchange_timestamp'] * 1000000) 
                    )
                sender.flush() # Flush batch to disk instantly
            except IngressError as e:
                print(f"QuestDB Write Error: {e}")
